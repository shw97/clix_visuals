    Getting Started

    This page contains doumentation of methodology and processes involved in generating visualisations of this website. We have a section for each type/super-set of visualisation, with all the information about data source, processing and different metrics used. As all the information related to every visualisation is in one page, it easier to navigate through them and see interlinks between different data visualisations.

    Tools Data Visualisation

    Data Source:

    Primary Source of tools data is json file generated whenever a user accesses Tools/Apps section of the CLix software. These files along with other thin data are available from schools as part of synthing data. In a given school (in a particular machine), seperate json file is generated for each tool. This json file will have logs of all the users of the tool on that machine.

    Data Processing:

    To process tools data, we first convert json files into csv files extracting only information relevant for our analysis. All these csv files of tools across all the schools is collated to create one big csv file. Each row of this csv file corresponds to a unique log of a user with the following features (columns):

    school_server_code: Unique code of the school in which log is generated
    user_id: Unique id of a student creating the log
    tool_name: Tool name of the tool being accessed
    created_at: Timestamp of creation of the log
    date_created: Date of creation of log
    time_spent: Time spent by a student(=user_id) on the tool(=tool_name) on a given day(=date_created). This is calculated by taking time difference between first and last log time_stamps of a every user_id in a given day.
    state_code: State in which machine is located

    Calculation of Metrics:

    Total Days Accessed

    This corresponds to total number of days tools section was accessed(irrespective of the time they spent) by unique students. In a given school, it is calculated by considering unique combinations of tool_name and user_id, then adding up number of unique date_created entries for each combination. This sum gives us the total number of unique days a tool was accessed by all users.

    Average Number of Students in a Day

    This gives us the number of students accessing the tools section of the platform in a day on an average. In a given school, for each unique combination of tool_name and date_created, we add up number of unique user_id's.

    Average Time Spent by a Student in a Day

    For a given school, all unique combinations of tool_name, user_id and date_created are considered, then for each of these time_spent is added up. This gives us the average time spent (averaged across students and days) for each tool.
